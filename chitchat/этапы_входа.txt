далее по порядку, разбираем фаил page.tsx 
у нас имеются функции getDeviceStatus() и getAuthStatus() 
папка lib находится на одном уровне что и папка app
page.tsx → вызывает getAuthStatus() из lib/store/auth/authStore.ts
page.tsx → вызывает getDeviceStatus() из lib/store/device/deviceStore.ts
функции getDeviceStatus() определяется первая когда пользователь 
заходит впервые в приложение и эти данные сохраняются с ключем deviceInfo
при повторных заходах в приложение эти данные не обновляются
    browserLanguage: "ru-RU" только языки
    deviceId: "jvmbsl"
    deviceType: "mobile" 
    platform: "Android"
    timezone: "Europe/Moscow"
    timezoneOffset: -180
    timezoneUTC: "UTC+3"
    userAgent: "Mozilla ..."
    ipAddress: "237.84.2.178"
дополнительно получаем IP адрес через "/api/device/userip" и сохраняем в переменную ipAddress

функция getAuthStatus() определяется вторая когда пользователь зашел 
первый раз и эти данные сохраняются с ключем authInfo по умолчанию
isAuthenticated = not_defined
isIdentifier = not_defined
isBlocked = not_defined
isExists = not_defined
isProtection = not_defined
isSecret = not_defined
isUserAgent = not_defined


/auth
├── open/                 → входная точка
├── blocked/              → если isBlocked
├── password/
│   ├── login/            → isExists === true
│   └── register/         → isExists === false
└── secret/
    ├── login/            → isProtection === true && isSecret === true
    └── register/         → isProtection === true && isSecret === false

    

во первых если пользователь зашел первый раз у него будет isBlocked = not_defined 
после процесса авторизации или регистрации у него уже будет проверен 
и у него будет установлен и статус будет уже либо true либо false, 
из этого следует такое условие:
если блокирован isBlocked = true → показываем страницу блокировки 
находится по пути app/(auth)/blocked/page.tsx


что нужно иметь ввиду, у нас есть HttpOnly куки
session_token (время жизни 60 минут) httponly
session_refresh (время жизни 7 дней) httponly
они будут после авторизации или регистрации
то есть пользователь уже ранее авторизованный, имеет эти куки
а если пользователь зашел первый раз, то у него нет этих куков

вопрос как нам сделать логику про куки?
